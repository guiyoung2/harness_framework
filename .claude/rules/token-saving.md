# 토큰 절약 가이드

CLAUDE.md에 명시된 5가지 절약 규칙의 부연 설명과, `@` 참조 가이드를 정리한다.

## 5가지 절약 규칙 (CLAUDE.md 본문 참조)

1. 이미 읽은 파일은 다시 읽지 않는다.
2. 도구 호출은 가능한 한 병렬로 실행한다.
3. 20줄 이상 분석은 서브에이전트에 위임한다.
4. 사용자가 이미 설명한 내용은 반복하지 않는다.
5. `@`로 500줄 초과 파일 전체 참조 금지.

## `@` 파일 참조 가이드

`@` 기호로 파일을 참조하면 해당 파일 내용 전체가 컨텍스트에 포함된다. 파일 크기별 권장 방법:

| 파일 크기 | 권장 방법 |
| --- | --- |
| 500줄 이하 | 전체 참조 가능 |
| 500 ~ 2,000줄 | 필요한 구간·심볼만 지정 |
| 2,000줄 이상 | 함수·클래스 단위로 분리해서 참조 |

## 컨텍스트 압축 타이밍

- 자동 압축은 컨텍스트 95% 시점에 실행되지만, 그때는 이미 품질이 저하된 상태다.
- 60% 시점에 수동으로 `/compact`를 실행하면 더 깔끔하게 압축된다.
- 압축 시 보존할 내용을 함께 지시한다: `/compact 실행해줘. 현재 구현 중인 X 기능과 발생한 에러는 보존해줘.`

## 셸 명령어 출력 주의

셸 명령어 실행 결과는 전체가 컨텍스트로 들어온다. 다음 패턴은 피한다:

- 커밋 200개짜리 `git log` — 필요한 범위만 (`--since`, `-n 20`)
- verbose 옵션 켠 전체 테스트 결과 — 모듈 단위로 분할
- 빌드 경고 가득한 로그 — 필요 시 `head`/`tail`로 상한선 설정

## MCP 서버 관리

MCP 서버를 연결하면 매 메시지마다 해당 서버의 도구 정의가 컨텍스트에 로드된다. 빈 엔터에도 토큰이 소모된다.

- `/mcp`로 현재 연결된 서버 확인
- 안 쓰는 서버 즉시 끊기
- 글로벌 vs 프로젝트 레벨로 분리: 글로벌에는 모든 프로젝트에서 항상 쓰는 것만, 그 외는 프로젝트 레벨로
